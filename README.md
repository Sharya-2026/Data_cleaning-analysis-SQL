# Data_cleaning-analysis-SQL
End-to-end PostgreSQL  project showcasing joins, CTEs, window functions, subqueries, and business insights from Airbnb listing, review, and calendar data
# 🏡 Airbnb Data Analysis using PostgreSQL

## Project Overview

This project presents an end-to-end SQL analysis of Airbnb listing data using PostgreSQL. The objective is to transform raw data into meaningful business insights by analyzing property listings, customer reviews, host performance, occupancy trends, pricing strategies, and estimated revenue. The project demonstrates how SQL can be applied to solve real-world business problems and support data-driven decision-making.

The database consists of three related tables—**Listings**, **Reviews**, and **Calendar**—connected through the `listing_id` field. A wide range of SQL techniques were used to explore and analyze the data, including table creation, joins, aggregate functions, subqueries, Common Table Expressions (CTEs), and window functions.

Key analyses include market size, average nightly price, room type distribution, occupancy analysis, neighbourhood performance, revenue estimation, host portfolio analysis, premium property identification, customer review analysis, and ranking of hosts and neighbourhoods. Estimated revenue was calculated using listing price and annual availability to provide business-focused insights.

### SQL Concepts Used

* DDL (CREATE TABLE, DROP TABLE)
* INNER JOIN and LEFT JOIN
* Aggregate Functions (COUNT, SUM, AVG)
* GROUP BY, ORDER BY, LIMIT
* Subqueries
* Common Table Expressions (CTEs)
* Window Functions (RANK, DENSE_RANK)

### Key Insights

* Total Airbnb listings and active hosts
* Average nightly prices across room types
* Most and least performing neighbourhoods
* Estimated revenue by room type, neighbourhood, and host
* Top-performing and low-performing hosts
* Most reviewed and premium listings
* Listings without customer reviews
* Top three most expensive listings in each neighbourhood

This project showcases practical SQL skills commonly required for Data Analyst roles, including relational database design, business reporting,
analytical query writing, and data exploration. It serves as a strong portfolio project demonstrating the ability to extract actionable insights from structured data using PostgreSQL.
